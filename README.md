# SE_BELARUS (временное название)

## О проекте

Микросервисная архитектура для управления товарами, услугами, вакансиями и пользователями.

## Структура проекта

| Микросервис | Описание | Порт |
|------------|----------|------|
| **sb_products** | Управление товарами (API + БД) | 8001 |
| **sb_services** | Управление услугами (API + БД) | 8002 |
| **sb_vacancies** | Управление вакансиями (API + БД) | 8003 |
| **sb_users** | Регистрация и авторизация пользователей (пароль, Google, Telegram) | 8004 |
| **sb_chats** | Настройки чата (требуется доработка фронтенда) | 8005 |
| **api_gateway** | Единая точка входа для всех микросервисов | 8006 |

## Запуск проекта

### Требования

- Docker & Docker Compose
- Python 3.11+

### Инструкция по запуску

1. **Клонирование репозитория**
   ```bash
   git clone <url-репозитория>
   cd SE_BELARUS
2. Создаем виртуальное окружение:
    ```
    python -m venv .venv
    .venv/Scripts/activate
   ```
3. Запускаем docker-compose.yml:
   
   Он установит все библиотеки и запустит все микросервисы. Единственное, с чем скорее всего будет проблема, это с созданием БД, но если что, ждите обновы:)
4. Все должно запуститься, после чего заходите в браузер на:
   ```http://localhost:8003/docs```
    Можете ещё посмотреть:
    ```
   http://localhost:8001/docs
   http://localhost:8002/docs
   http://localhost:8005/docs
   http://localhost:8006/docs
   http://localhost:8007/docs
   ```
   
    Это локальные адреса API остальных микросервисов. В дальнейшем к ним закроется доступ со стороны пользователя, останется доступ только через api_gateway.
5. Добавить в каждый микросервис файл .env (его содержимое скину до 10.04.2026 в Telegram, базарю);
6. Если всё нормас, то всё нормас, если нет, то это ваши проблемы - я за это деньги не получаю:)
